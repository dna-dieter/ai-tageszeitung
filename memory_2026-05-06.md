# Memory für nächsten Agenten — AI Tageszeitung
**Stand: 2026-05-06**

## Projekt-Kontext
Automatische Online-Tageszeitung für zwei NRW-Kreise, deployed via GitHub Pages.
- **Repo**: dna-dieter/ai-tageszeitung
- **Live-Site**: GitHub Pages des Repos
- **Kreise**: Rheinisch-Bergischer Kreis (8 Kommunen) + Oberbergischer Kreis (13 Kommunen)
- **Update-Mechanismus**: GitHub Actions Workflow (`update.yml`), läuft stündlich um :05 plus `workflow_dispatch` (manuell)

## Architektur-Kurzüberblick
- `scripts/config.py` — zentrale Kreis/Kommune-Definitionen + Keywords für Klassifikation
- `scripts/fetch_feeds.py` — holt RSS-Feeds, klassifiziert, speichert JSON
- `scripts/build_html.py` — baut HTML-Seiten in `docs/`
- `<kreis>/feeds.json` — RSS-Feed-Liste pro Kreis (mit `verified`, `coverage`, optional `pages`)
- `<kreis>/<kommune>/articles.json` — Artikel je Kommune
- `<kreis>/all_articles.json` — kompletter Archiv-Index je Kreis
- `docs/` — generiertes HTML (wird committed, bedient GitHub Pages)

## Wichtige Konventionen
- **Feeds** haben `coverage: "<kommune-slug>"` (Direkt-Zuweisung) oder `"kreisweite-nachrichten"` (dann wird via Keywords klassifiziert)
- **Keywords** stehen in `config.py`, case-insensitive Substring-Match auf Titel+Summary
- **Dedup** via MD5(link+title), alte Artikel bleiben erhalten (inkrementell)
- **Pagination** bei WordPress-Feeds via `pages: N` (WordPress `?paged=N`). Der Fetcher hängt `&paged=N` wenn die URL schon `?` enthält, sonst `?paged=N`.

## Tricks / Praktisches
- **Workflow von hier triggern**: `gh workflow run update.yml --repo dna-dieter/ai-tageszeitung`
- **Latest Run Status**: `gh run list --workflow=update.yml --repo dna-dieter/ai-tageszeitung --limit 1`
- **Lokal den Workflow simulieren**: `cd scripts && python3 fetch_feeds.py && python3 build_html.py`
- **RSS-Feed testen**: `curl -sL -A "Mozilla/5.0" URL | grep -c '<item>'`
- **WordPress-Custom-Post-Types entdecken**: Startseite der Gemeinde anschauen, Pfad der Artikel-Links ansehen (`/aktuelle-ereignisse/...` → `post_type=aktuelle-ereignisse`)

## Bekannte Probleme
- **RGA-Kreis-Unterfeeds** (z.B. RGA Wipperfürth): Manche liefern leeren Channel — RGA-seitig deaktiviert, kein Fix möglich. Der RGA-Gesamt-Feed funktioniert aber.
- **Ältere `.DS_Store`-Einträge** im Git-Log: Harmlos, `.gitignore` verhindert neue.
- **Pagination-Caveat**: Der Fetcher bricht bei erster Seite ohne Items ab (`break`). Bei flaky Feeds mit zwischendrin leerer Seite würden nachfolgende Seiten nicht mehr geholt. Das hat sich bisher nicht als Problem gezeigt.
